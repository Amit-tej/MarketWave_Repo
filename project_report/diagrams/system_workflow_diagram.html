Figure 5.1: MarketWave System Workflow – Input to Output

INPUT
Raw Agricultural Data
PREPROCESSING
Data Cleaning & Preparation
PROCESS
Feature Engineering
INTERNAL PROCEDURE
ML Model Ensemble
OUTPUT
Price Predictions

🔹 INPUT Stage

  • Raw CSV files from CEDA (2021–2024)
  • Price data: p_min, p_max, p_modal
  • Market info: commodity, variety, market_name
  • Location: state_name, district_name
  • Temporal: date, arrival information

🔹 PREPROCESSING Stage

  • Date column detection and parsing
  • Handling missing values (ffill/bfill/interpolate)
  • Outlier removal using IQR method
  • Price column normalization
  • Filtering Telangana/Hyderabad data

🔹 PROCESS Stage (Feature Engineering)

  • Temporal features: day, week, month, quarter, year
  • Lag features: 1-day, 7-day, 14-day, 30-day
  • Rolling statistics: mean, std, min, max, range
  • Momentum & RSI indicators
  • Fourier terms for seasonality
  • Target encoding for categorical variables

🔹 INTERNAL PROCEDURE (ML Pipeline)

  • Three base models: XGBoost, Prophet, LSTM
  • Individual model training per market/commodity
  • Meta-learner (Ridge regression) for stacking
  • Conformal prediction intervals (80%, 90%, 95%)
  • Fallback to statistical predictor if ML fails

🔹 OUTPUT Stage

  • 30-day price forecasts (yhat values)
  • Confidence intervals at 80%, 90%, 95% levels
  • Individual model predictions (base_preds)
  • Current market price from web scraping
  • Interactive dashboard visualization
